## 树的基本概念

树是 $n$ 个结点的有限集合，具有唯一的根结点且子树互不相交

**树的专业术语**

1. 结点的度为子树的个数，树的度为各结点度的最大值
2. 叶子结点即度为 0 的结点，分支结点即度大于 0 的结点
3. 结点的层次从根开始定义，树的高度或深度为结点的最大层数
4. 路径长度指路径上所经过的边的数目

## 二叉树的逻辑与存储

二叉树是每个结点最多只有两个子树，且子树有左右之分的树结构

**特殊二叉树**

1. 满二叉树：所有分支结点都存在左、右子树，且叶子都在同一层
2. 完全二叉树：叶子结点只可能在层次最大的两层出现，且结点编号与满二叉树一致

**二叉树的性质**

1. 在二叉树的第 $i$ 层上至多有 $2^{i-1}$ 个结点
2. 深度为 $k$ 的二叉树至多有 $2^k-1$ 个结点
3. 对任何二叉树，若叶子结点数为 $n_0$，度为 2 的结点数为 $n_2$，则 $n_0 = n_2 + 1$

**存储结构**

1. 顺序存储结构：用一组地址连续的存储单元存储，仅适用于完全二叉树
2. 二叉链表存储：结点包含数据域、左孩子指针域和右孩子指针域

## 二叉树的遍历与线索化

遍历是按某种搜索路径访问二叉树中所有结点且仅访问一次的过程

**遍历算法**

1. 先序遍历：根、左、右
2. 中序遍历：左、根、右
3. 后序遍历：左、右、根
4. 层次遍历：利用队列按层从左至右访问
5. 算法名称：递归遍历算法、非递归遍历算法、层次遍历算法

**线索二叉树**

1. 利用二叉链表中的空指针域存放指向前驱或后继的线索
2. 若左指针为空则指向前驱，若右指针为空则指向后继
3. 算法名称：二叉树线索化算法、在线索树中找前驱后继算法

## 树、森林与哈夫曼树

**树与森林的转换**

1. 树转换为二叉树：左孩子右兄弟法
2. 森林转换为二叉树：将各树转换后的二叉树根结点视为兄弟关系连接

**哈夫曼树（最优二叉树）**

1. 带权路径长度（WPL）达到最小的二叉树
2. 构造原则：每次选取权值最小的两个结点作为左右子树构造新树
3. 算法名称：哈夫曼树构造算法、哈夫曼编码算法

## 题目索引

**二叉树相关题目**

1. **计算二叉树结点总数**：利用递归统计所有结点个数

2. **计算叶子结点数**：递归判断左右孩子皆为空的结点

3. **求二叉树深度**：递归比较左、右子树深度并取最大值加 1

4. **复制二叉树**：递归创建新结点并生成副本

5. **判断二叉树相似/相等**：递归比较两棵二叉树的结构与值

6. **交换二叉树左右子树**：递归地交换每个结点的左右指针

7. **统计度为1的结点**：遍历并计数仅有一个孩子的结点

8. **非递归中序遍历**：利用栈实现不使用递归的树遍历

9. **由遍历序列构造二叉树**：根据先序+中序或后序+中序恢复树结构

10. **二叉树宽度计算**：利用层次遍历统计结点最多的层

**一般树与森林相关题目**

1. **树的叶子结点统计**：在孩子兄弟表示法下统计叶子数

2. **计算树的高度**：基于孩子兄弟存储结构递归求解

3. **森林的遍历**：实现森林的先序和中序遍历逻辑

4. **哈夫曼编码生成**：根据给定权值集合构造编码表

5. **并查集操作**：实现树结构的查找根结点与合并子树算法
